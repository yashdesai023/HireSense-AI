🗄️ Database Schema Overview

  • 👤 User Management: Core user entity with role-based access control
  • 💼 Job Posting: Job listings with recruiter associations
  • 📎 Resume Processing: Resume uploads with AI-parsed skills
  • 🧪 Assessment System: Tests, questions, and scoring mechanism
  • 📋 Application Tracking: Job applications with status management
  • 🎯 Interview Scheduling: Interview coordination and feedback

🔗 Key Relationships:

  • User → Job: Recruiters post jobs (one-to-many)
  • User → Resume: Candidates upload resumes (one-to-one)
  • User → Application: Candidates apply to jobs (one-to-many)
  • User → Score: Candidates take tests (one-to-many)
  • Job → Test: Jobs have associated skill tests (one-to-many)
  • Test → Question: Tests contain multiple questions (one-to-many)
  • Job → Interview: Jobs schedule interviews (one-to-many)

🔐 Security Features:

  • Password Hashing: Secure password storage
  • Role Enumeration: Admin, Recruiter, Candidate roles
  • Foreign Key Constraints: Data integrity enforcement
  • Unique Constraints: Email uniqueness and data validation
